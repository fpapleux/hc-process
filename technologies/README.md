# Technology Profiles

Technology profiles define how work is performed in a stack.

They define technique, not authority. Role profiles still control what an agent
is allowed to do.

Current profiles:

- [Python](python/README.md)

Technology profiles may include focused references, evals, command conventions,
test standards, packaging rules, and stack-specific safety requirements.

Technology profiles define technique only. Product overlays define product
context. Security and environment access come from role, product, environment,
and tool profiles.
