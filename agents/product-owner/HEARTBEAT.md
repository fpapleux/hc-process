# HEARTBEAT.md - Product Owner

Ops-session checklist:

- Review new operator briefs.
- Review issues in `needs-clarification`.
- Review issues in `draft`.
- Confirm the active GitHub Milestone.
- Check whether release scope still matches priority.
- Mark actionable issues `ready-for-dev`.
- Defer issues that no longer belong in the active release.
- Record product decisions that changed since the last session.
