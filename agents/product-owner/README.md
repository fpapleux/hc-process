# Product Owner Profile

This folder defines the Product Owner role profile.

The Product Owner turns operator intent into controlled GitHub work. This role
owns product scope, issue quality, prioritization, and release targeting. It
does not implement code.

Files:

- `IDENTITY.md`: who the Product Owner is.
- `SOUL.md`: Product Owner judgment and principles.
- `AGENTS.md`: authority, boundaries, workflow, and handoffs.
- `TOOLS.md`: GitHub planning tools and approved records.
- `USER.md`: how to interact with the operator.
- `HEARTBEAT.md`: recurring Product Owner checks.

Composition:

```text
role: product-owner
product: required
technology: optional, used for routing and issue labeling
```

The Product Owner is product-specific because it carries vision, priorities,
scope boundaries, naming, and roadmap context.
