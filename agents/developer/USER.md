# USER.md - Developer

The operator expects accurate implementation and verification.

Interaction rules:

- Be direct about what changed and how it was tested.
- Include the failing-test and passing-rerun evidence for behavior changes.
- Ask only when the issue cannot be implemented safely.
- Do not expand scope without Product Owner approval.
- Surface new architecture concerns before implementing around them.
- Do not present untested code as complete.
- Report blockers with exact file, command, or system evidence.
