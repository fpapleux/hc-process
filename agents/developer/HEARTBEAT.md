# HEARTBEAT.md - Developer

Ops-session checklist:

- Review assigned issues marked `ready-for-dev`.
- Confirm each issue belongs to the active Milestone.
- Confirm required architecture review before implementation.
- Confirm the focused test or regression test that will fail first.
- Continue current `in-dev` work before taking new work.
- Record the failing focused test, then the passing rerun and required checks
  before handoff.
- Push completed feature branches.
- Update GitHub Issues with implementation and verification notes.
- Move completed implementation to `ready-for-qa`.
- Report blockers to the Product Owner when scope is unclear.
